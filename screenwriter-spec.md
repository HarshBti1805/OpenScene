# PART 1: MARKET RESEARCH

## 1.1 Final Draft 13

**Position:** The industry standard. Used by roughly 95% of film/TV professionals. macOS and Windows only. One-time purchase, $199 to $250 full license, ~$99 upgrades. No free tier (30-day trial only).

**Core features:**
- Automatic industry-standard formatting and pagination (the gold standard for page-count accuracy)
- Beat Board (visual corkboard for brainstorming) and Outline Editor with Structure Lines
- Navigator 2.0: scene list, character screen time tracking, character interaction tracking, inclusivity/gender stats, outline editing directly in the navigator
- Revision Mode: colored revision pages (industry production standard: blue, pink, yellow, etc.), locked pages, revision marks (asterisks)
- ScriptNotes, tracked changes, commenting, automatic version saving
- Collaboration (real-time co-writing, requires internet)
- Speech-to-script, text-to-speech table reads with assignable character voices
- Templates: screenplays, teleplays (with real show templates), stage plays, comics, graphic novels, VR/immersive
- Tagging for production breakdowns; export to scheduling/budgeting software (Movie Magic ecosystem)
- Night/Midnight/Typewriter modes, writing goals and productivity stats
- FDX is its native format and the de facto interchange format of the industry

**Free vs paid:** Everything is paid. Nothing meaningful is free.

**What users value most:** Bulletproof pagination and formatting trusted by production offices; revision workflow (colored pages, locked pages, A-scenes); the fact that "send me the Final Draft file" is an industry sentence; Beat Board + Navigator for structure.

**Common complaints:** Price; slow release cadence; cosmetic updates charged as paid upgrades; sign-in/activation requirements; no Linux support.

## 1.2 Fade In

**Position:** The professional's budget pick. $79.95 one-time purchase with free lifetime updates. Mac, Windows, **and Linux**. Used by pros like Craig Mazin and Rian Johnson (Knives Out).

**Core features:**
- Full industry-standard formatting and accurate pagination
- Revision management: revision marks, colored revision sets, locked pages, compare drafts
- Scene navigator, index cards, outlining
- Character/location autocompletion, character name management (global rename)
- Dialogue tuner (view all dialogue for one character in sequence)
- Watermarking, production reports (scene reports, cast reports, location reports)
- Full FDX import/export, Fountain support, PDF, HTML, Celtx, Trelby import
- Dual dialogue, alternate dialogue lines, ScriptNotes
- Full-screen minimal mode, dark mode, typewriter scrolling
- Multi-language spell check; solid Unicode support

**Free vs paid:** Free demo has every feature but watermarks PDF/print output. The $79.95 license unlocks clean export. No subscription, no cloud requirement.

**What users value most:** Stability and speed ("less buggy than Final Draft" is a recurring sentiment); one-time price with free updates forever; Linux support; complete revision/production toolset at a fraction of Final Draft's price; clean, no-nonsense UI.

## 1.3 Arc Studio Pro

**Position:** The modern, design-forward newcomer. Free tier + Essentials ($69/yr) + Pro ($99/yr). Web, macOS, Windows, iOS.

**Core features:**
- Distraction-free, beautifully designed editor with industry formatting
- Plot Board / beat outlining with drag-and-drop, beats linked to scenes, automatic color coding
- Real-time collaboration with offline-capable editing and conflict-free syncing
- Comments with tagging, filtering, resolving; feedback links for reviewers
- Revision tracking, restore deleted text, script versions/snapshots
- Table Read (assigned AI voices, downloadable audio), character image generation
- Writing sprints, reminders, writing goals
- Templates (20+, film/TV/stage)
- Automatic cloud + device backup

**Free vs paid:**
- Free: 2 scripts, core editing, commenting, collaboration participation, watermarked PDF export
- Essentials ($69/yr): unlimited scripts, watermark-free export, advanced formatting
- Pro ($99/yr): offline desktop apps, outlining/Plot Board, inviting collaborators, custom formats, revision management

**What users value most:** The plot-board-to-script linkage (outline beats stay attached to scenes); clean minimal UI; frictionless commenting/feedback workflow; "it just looks and feels modern."

**Complaints:** Subscription fatigue; offline mode locked behind the top tier; pagination fidelity questioned by some pros.

## 1.4 Highland 2 / Highland Pro

**Position:** John August's Fountain-native, Mac-only minimalist tool. Highland 2 was $49.99 one-time (free with watermark); the line has consolidated into Highland Pro at ~$4.99 to $9.99/month. Beloved by writers like Phil Lord and Michael Chabon.

**Core features:**
- Plain-text Fountain writing: type naturally, the app infers scene headings, characters, dialogue
- Live preview of formatted pages; export to PDF and FDX
- Navigator (document structure sidebar)
- The Bin: a scratch area to drag scenes/text out of the script without deleting them
- Writing Sprints (Pomodoro-style timer with word targets)
- Revision Mode
- Gender Analysis: dialogue and word counts broken down by character gender (widely praised, genuinely unique)
- PDF "melting": converts screenplay PDFs back into editable text
- Templates (stage play, multicam, graphic novel, novel, treatment, etc.) and themes (paid)
- Also handles Markdown prose, so novels/treatments live in the same tool

**Free vs paid:** Free version fully functional but watermarks exports; Pro unlocks watermark-free PDF, templates, themes, Sprint, Revision Mode (exact gating has shifted across versions).

**What users value most:** Speed and calm ("moves at the speed of thought"); Fountain plain-text ownership of your work forever; the Bin; Gender Analysis; PDF melt.

**Complaints:** Mac-only; shift from one-time purchase to subscription; light on production features.

## 1.5 Celtx

**Position:** The all-in-one pre-production suite (script + storyboards + breakdowns + scheduling + budgets + call sheets). Cloud-based, subscription: free tier, then roughly $7.50 to $25/month depending on plan and billing.

**Core features:**
- Multi-format script editor (film, TV, stage, podcast, game/interactive, documentary AV scripts)
- Real-time collaboration, commenting
- Index cards, outlining, beat sheets
- Script breakdown with tagging (cast, props, wardrobe, locations)
- Storyboarding, shot lists, scheduling, budgeting, call/sync sheets, catalogs
- Cloud storage and multi-device access

**Free vs paid:** Free: one active project, basic editor, PDF export. Almost everything else (multiple projects, breakdowns, storyboards, scheduling, most exports, full collaboration) is paid.

**What users value most:** Writing-to-production continuity in one tool (indie filmmakers who wear every hat); the AV/two-column and game script formats that pure screenplay tools lack.

**Complaints:** Writing experience has become cluttered as the product pivoted toward production teams; formatting fidelity not quite Final Draft-grade; aggressive paywalling.

## 1.6 WriterDuet

**Position:** The collaboration king. ~6 million users. Web-first with desktop/mobile apps. Free: 3 scripts. Plus ~$7.99 to $9.99/mo, Pro $11.99/mo or $89/yr, Premium ~$13.99 to $15.99/mo. Lifetime licenses periodically offered.

**Core features:**
- Real-time co-writing (Google Docs-grade concurrency, the best in class)
- Time Machine: edit-by-edit history with author attribution and playback
- Ghost Mode: write privately, then reveal to collaborators
- Built-in text/video chat
- Cards, mind maps, outlining, sequences
- Revision tracking, line locking, tagging/filtering, production tools
- Statistics and reports, document analytics
- Read-aloud with computer voices, dictation, translation (Premium)
- Parallel columns (split screen / VR) (Premium)
- Huge import/export range: FDX, Fountain, Celtx, PDF, DOCX
- Offline desktop app (paid tiers)

**Free vs paid:**
- Free: 3 scripts, no page limits, full import/export, industry templates, cloud backup, can participate in collaboration
- Paid: unlimited scripts, full revision history, offline apps, tagging, stats/reports, voices, columns, inviting collaborators

**What users value most:** Real-time collaboration that actually works; Time Machine; generous free tier for getting started; cross-device ubiquity.

**Complaints:** Performance/slowness in big documents; feature gating spread across four confusing tiers; cloud dependence.

## 1.7 Scrivener

**Position:** Not a screenwriting tool per se, but a long-form writing IDE ($49 to $59 one-time, macOS/Windows/iOS) with a scriptwriting mode. Used by novelists who also write scripts.

**Core features:**
- Binder (hierarchical project organization), corkboard, outliner
- Research storage inside the project (PDFs, images, web pages, notes)
- Scriptwriting mode with screenplay element formatting; FDX and Fountain export
- Snapshots (per-document version history), split view, composition (focus) mode
- Compile system: one manuscript, many output formats
- Keywords, labels, statuses, metadata, collections/saved searches
- Writing targets and statistics

**Free vs paid:** One-time purchase; 30-day trial. No subscription.

**What users value most:** Everything-in-one-project organization (research + notes + drafts); corkboard/outliner; the price model.

**Complaints for screenwriting specifically:** Formatting/pagination is not production-grade; no revision marks/colored pages; screenplay features feel bolted on; learning curve.

## 1.8 KIT Scenarist (and its successor Story Architect)

**Position:** Free, open-source, cross-platform (Windows/macOS/Linux/Android). The closest existing thing to what this spec proposes. Desktop is fully free; optional cloud sync is the only paid part (~$4.99/mo).

**Core features:**
- Four modules: Research (characters, locations, documents, images, mind maps), Cards (corkboard), Script editor (auto-formatting, multiple national formats), Statistics
- Detailed statistics: character activity across the story, scene dynamics, dialogue counts, reports
- Import: FDX, Fountain, Celtx, Trelby; Export: PDF, FDX, Fountain, DOCX
- Multi-language script support and localized UI
- Scene numbering, script templates

**Free vs paid:** All features free on desktop; only cloud storage/sync for collaboration costs money.

**What users value most:** The Research module (a real story bible inside the project); the Statistics module (unmatched among free tools); true cross-platform including Linux; no watermarks or limits.

**Complaints:** Bugs/polish gaps typical of a small open-source project; transitional instability while moving to Story Architect; dated UI; performance.

## 1.9 Trelby

**Position:** Fully free, open-source, Windows/Linux (no native Mac). Minimal, fast, but effectively dormant for years.

**Core features:**
- Automatic screenplay formatting, fast plain UI
- Scene/character navigation, name database (character name suggestions)
- Reports: scene, location, character, dialogue reports
- Compare drafts
- Import/export: FDX, Celtx, Fountain, PDF (built-in PDF generator), formatted text
- Watermarked PDF generation for distribution drafts

**Free vs paid:** 100% free. No paid anything.

**What users value most:** Zero cost, zero clutter, works offline forever, code is auditable and forkable.

**Complaints:** Long-abandoned development; dated UI; no Mac build; no outlining/index cards; English-centric.

## 1.10 Synthesis: What Users Consistently Value Most (across all tools)

1. **Flawless industry-standard formatting and pagination.** Non-negotiable. A page must equal a page. This is why Final Draft survives at 3x the price of rivals.
2. **The editor gets out of the way.** Speed, distraction-free modes, Tab/Enter element cycling, autocomplete of character names and locations. Highland's entire brand is built on this.
3. **File ownership and portability.** Fountain's popularity, hatred of watermarks, resentment of subscriptions and cloud lock-in. Writers fear losing access to their own work.
4. **FDX interchange.** Whatever you write in, you must be able to hand over an FDX and receive one back.
5. **Structure tools linked to the script.** Beat boards, index cards, outlines that stay synchronized with scenes (Arc Studio's plot board, Final Draft's Beat Board).
6. **Revision workflow.** Colored revision pages, locked pages, revision marks, draft comparison. This is where free tools fall short of Fade In/Final Draft and where production offices draw the line.
7. **Version safety.** Automatic snapshots, "time machine" history, restore deleted text. WriterDuet's Time Machine is one of its most loved features.
8. **Reports and statistics.** Character screen time, dialogue counts, gender/inclusivity analysis (Highland's Gender Analysis and Final Draft's inclusivity stats are frequently praised), scene/location reports for production.
9. **Working offline.** Recurring complaint against Arc Studio/Celtx/WriterDuet free tiers; recurring praise for Fade In, Highland, KIT Scenarist, Trelby.
10. **Fair pricing.** One-time purchase (Fade In) or genuinely free (KIT, Trelby) earns intense loyalty; subscriptions and paywalled offline modes earn resentment.

**The market gap this product fills:** No tool today combines (a) truly free and open source, (b) modern, beautiful UI, (c) production-grade formatting and revision features, (d) full offline-first local storage, and (e) all three desktop platforms. KIT Scenarist is closest but is dated and rough; Fade In is closest in features but closed-source and paid; Arc Studio is closest in UX but cloud-first and subscription-gated.

# PART 2: PRODUCT SPECIFICATION

Design principles that govern every feature below:

- **Completely free, forever.** 100% of features are free on all platforms. No paid tiers, no pro version, no watermarks, no trials, no accounts, no license keys, no feature gating of any kind. Licensed under a permissive open-source license (MIT or Apache-2.0). There is no cost surface anywhere in the product.
- **No AI. Ever, by design.** The product contains no generative AI features and reserves no scope for them: no AI writing, feedback, summarization, or generation of any kind, in core or in plugins shipped by the project. This is a stated non-goal, not a deferred feature. It is also a positioning asset: writers with WGA/authorship concerns get a guarantee that nothing in the tool touches their words, and the plugin API's terms prohibit first-party AI extensions so the promise can't erode over time.
- **Local-first, always.** Every feature works with zero network access. The app never requires an account, sign-in, or license check.
- **Your files, readable forever.** The native project format is documented, plain-text-based, and human-recoverable.
- **Professional-grade output.** PDF and pagination quality must match Final Draft/Fade In, not "good enough for a student film."
- **Calm by default, powerful on demand.** The default view is a page and a cursor. Everything else is discoverable, dockable, and dismissible.
- **Priority labels:** **MVP** (v1.0 launch), **v1.x** (fast-follow within the first year), **v2+** (later major release).

## 2.20 Collaboration and Teams (Free, No Vendor Cloud)

The collaboration model is layered. Each layer is fully free and adds capability without ever introducing a vendor server, an account requirement, or a fee. Layers 1 and 2 need nothing but files; Layer 3 works on a local network; Layer 4 uses infrastructure the team hosts itself.

**Layer 1: File-based feedback (asynchronous, zero infrastructure)**

| Feature | What it does | Why it matters | Priority |
|---|---|---|---|
| Review-copy round trips | Export a read-only "review copy" file; reviewers open it in the free app, add comments and margin notes, and send the file back over email/WhatsApp/drive; comments merge into the master with reviewer attribution. | Replicates 80% of cloud feedback workflows with plain files and no server at all. | v1.x |
| Comment threading and resolution | Merged comments support replies, categories, and resolve/reopen states. | Feedback is a conversation, not a list; resolution state tracks the rewrite. | v1.x |

**Layer 2: File-based co-writing (asynchronous, zero infrastructure)**

| Feature | What it does | Why it matters | Priority |
|---|---|---|---|
| Draft merge assistant | Import a collaborator's edited copy; three-way merge UI at scene/line level with conflict resolution and preview. | Co-writing over any channel (email, USB, synced folder) without a service. | v2+ |
| Sync-folder co-writing | Because documents are CRDT-backed (see below), two writers pointing at the same Dropbox/Syncthing/Google Drive folder can edit at different times and changes merge automatically without conflict-copy chaos. | Turns storage the user already has into the sync layer; the app never needs its own cloud. | v2+ |
| Git-based workflow support | Plain-text Fountain-superset files diff and merge cleanly in git; the app detects a git repo and can surface commit/branch state for technical teams. | Free, battle-tested infrastructure for versioned team writing at zero cost. | v2+ |

**Layer 3: Real-time co-writing (peer-to-peer)**

| Feature | What it does | Why it matters | Priority |
|---|---|---|---|
| CRDT document core | Script documents are backed by a CRDT (conflict-free replicated data type), so any two copies of a document can always merge automatically and deterministically, regardless of how the edits traveled. | The single architectural decision that makes every collaboration feature above and below possible without a central server. Built into the document model from v1 even though multi-user features ship later. | **MVP** (data model), features v2+ |
| LAN session | One writer hosts a session; others on the same network discover and join (mDNS/QR code); live cursors, presence, and simultaneous editing with per-author color. | Writers' rooms sit in one room on one Wi-Fi network; this delivers WriterDuet-grade live co-writing with literally zero infrastructure. | v2+ |
| Direct P2P over internet | Remote co-writing via WebRTC data channels: peers connect directly; only a lightweight open signaling handshake is needed to find each other (community-hosted or self-hosted, interchangeable). | Two remote co-writers collaborate live without anyone running a real server. | v2+ |
| Ghost drafting | Write privately within a live session, reveal when ready. | WriterDuet's beloved Ghost Mode; protects creative safety in rooms. | v2+ |
| Session chat | Ephemeral text chat inside a live session (never stored in the script file). | Keeps room chatter out of email; expected in live tools. | v2+ |

**Layer 4: Teams (self-hosted, optional, free)**

| Feature | What it does | Why it matters | Priority |
|---|---|---|---|
| Slate Relay (self-hosted server) | A tiny open-source server (single binary / Docker image) any team can run on a $5 VPS, an office machine, or a Raspberry Pi. It does three things: relays real-time sessions for remote teams (when P2P is blocked by NATs), stores encrypted project replicas for always-available sync, and hosts the team roster. The desktop app is a full product without it. | Gives distributed teams the "always-on shared project" experience of Arc Studio/WriterDuet with no vendor and no fee; ideologically consistent (the team owns its server and its data). | v2+ |
| Team workspaces | A relay hosts named workspaces containing shared projects; members see a shared project list and open/sync projects from it. | The "team drive" experience production companies and writing staffs expect. | v2+ |
| Roles and permissions | Per-project roles: Owner, Writer (edit), Commenter (notes only), Reader (view/print only, optionally watermarked per user). | Rooms include showrunners, staff writers, assistants, and execs with very different rights; watermarked reader copies deter leaks. | v2+ |
| Author identity and attribution | Local identity (name + key pair, no account service); every edit, comment, and revision mark carries verified authorship into history and Time-Machine playback. | "Who wrote this line" is a daily writers' room question and occasionally a credit-arbitration one. | v2+ (identity primitive at MVP in data model) |
| Presence and activity | See who has a project open, who's in which scene, and a team activity feed (drafts exported, versions named, comments added). | Coordination glue for staffs; cheap once the relay exists. | v2+ |
| End-to-end encryption option | Project content encrypted client-side so even a shared/rented relay host can't read scripts. | Studios and paranoid writers alike; strong differentiator vs. cloud incumbents. | v2+ |
| Episode/room workflows | Team templates: shared series bible, per-episode assignments, draft status board across a season. | Maps the tool onto how TV staffs actually organize work. | v2+ (after series management) |

**Why this model works while staying free:** every layer either uses no infrastructure at all (files, LAN, P2P) or infrastructure the team already has or can trivially self-host (a synced folder, a git repo, a tiny relay binary). # PART 3: TECHNOLOGY STACK RECOMMENDATION

## Recommended stack: Tauri 2 + React + TypeScript, with a Rust core

| Layer | Choice | Rationale |
|---|---|---|
| Application shell | **Tauri 2** | Native performance profile: ~10x smaller binaries and a fraction of Electron's memory, using each OS's native webview. First-class macOS/Windows/Linux support (Linux is a real target, not an afterthought). Security-sandboxed by design, which matters later for the plugin system. |
| UI | **React 19 + TypeScript + Tailwind CSS** (shadcn/ui-style component approach) | The strongest possible fit for AI-assisted development with Claude: React/TS/Tailwind is the stack Claude generates most reliably and idiomatically, which directly maximizes vibecoding velocity and maintainability. Also the largest contributor pool for an open-source project, and the easiest path to a genuinely beautiful, custom-designed UI. |
| Editor engine | **Custom editor built on ProseMirror** (or CodeMirror 6 if a text-first editing model is preferred) | Screenplay editing is a structured document problem: ProseMirror's schema model maps perfectly to typed elements (scene heading, action, dialogue), gives robust undo history, decorations (revision marks, notes), and collaborative-editing primitives (CRDT/OT-ready for the v2 LAN feature). Off-the-shelf editors can't do screenplay pagination; a custom layout/pagination layer on top is unavoidable in any stack, and ProseMirror is the best-proven foundation for it. |
| Core logic (heavy lifting) | **Rust (Tauri backend)** | Pagination engine, FDX/Fountain parsers, PDF generation, diffing/snapshots, search indexing, and file integrity all run natively in Rust: deterministic, fast on 200-page documents, and memory-safe. This is what makes "instant typing at scale" achievable where pure-JS apps (WriterDuet) struggle. |
| Document format | **Fountain-superset plain text + a sidecar JSON/SQLite** per project | Script bodies stored as extended Fountain (human-readable forever, git-diffable, honoring the ownership promise); structured metadata (scene data, characters, notes, snapshots index) in SQLite via Rust. Best of both: readable files + fast queries. |
| Local data | **SQLite** (bundled, via Rust) | Zero-config, battle-tested, perfect for snapshots/history/search index/stats; single-file, backup-friendly. |
| PDF generation | **Rust-native PDF rendering** (printpdf/typst-style pipeline driven by the same layout engine as the editor) | One layout engine drives screen, print, and PDF, guaranteeing "what you see is what Final Draft sees." Avoids headless-browser PDF fragility. |
| State management | **Zustand** (+ React context) | Minimal, predictable, easy for contributors and for AI-assisted edits. |
| Testing | Vitest + Playwright (UI), Rust unit/property tests for parsers and pagination (golden-file tests against reference PDFs) | Pagination correctness must be regression-locked; golden-file testing against FDX/PDF references is the industry-fidelity guarantee. |
| Plugins (v2) | JS plugin runtime in a sandboxed webview/worker with a typed API | Web-tech plugins recruit the largest developer pool; Tauri's permission model contains them. |
| Collaboration core (v2) | **Loro or Automerge (Rust CRDT)** embedded in the document model; **WebRTC** data channels for P2P sessions; **mDNS** for LAN discovery; **Slate Relay** as a small Rust (axum) binary with SQLite, shipped as a single executable and Docker image | Rust-native CRDTs merge in the core where the document lives; WebRTC avoids servers for live sessions; the relay stays trivially self-hostable (one binary, one file DB), which is what keeps team features free. |

## Why not the alternatives

- **Electron:** Same web-UI benefits, but 150MB+ binaries, heavy RAM, slower startup, and a worse story for the "fast and lean" brand this product needs. Chromium bundling also complicates Linux packaging. Choose it only if pixel-identical webviews across OSes outweigh everything (they don't here; the layout engine is custom anyway).
- **Flutter:** Great rendering consistency, but custom text-editing at this depth (structured elements, IME correctness, screen readers, pagination) is far harder in Flutter than on the web platform, desktop accessibility is weaker, and Claude-assisted productivity in Dart trails TypeScript significantly.
- **Qt/QML (C++):** Maximum native performance and what KIT Scenarist uses, but slowest development velocity, smallest OSS contributor pool, licensing friction, hardest for AI-assisted development, and hardest to make beautiful.
- **Native per-platform (Swift/WinUI/GTK):** Triple the work, triple the drift; ruled out by a small team and the cross-platform requirement.

## How the stack maps to the priorities

- **Native desktop performance:** Rust core for parsing/pagination/PDF; native webviews; virtualized page rendering. Performance budget: <2s cold start, <16ms keystroke-to-paint at 250 pages.
- **Beautiful UI:** Full web styling freedom (Tailwind + custom design system), custom typography and theming, none of the "toolkit look."
- **Offline-first, local storage:** No server anywhere in the architecture; Fountain + SQLite on disk; Tauri apps are fully functional without network by construction.
- **Easy AI-assisted development:** React/TS/Tailwind + well-documented Rust commands is the highest-leverage stack for Claude-driven implementation, and the strict UI/core split keeps generated code reviewable.
- **Maintainability:** Typed end to end (TS + Rust), small dependency surface, one layout engine, golden-file tests guarding the crown jewel (pagination).
- **Cross-platform:** Tauri 2 ships signed builds for macOS (universal), Windows (MSI/exe), and Linux (AppImage/deb/rpm/Flatpak) from one codebase.

**Primary risk and mitigation:** Linux webview (WebKitGTK) inconsistencies are Tauri's known weak spot. Mitigation: the custom layout engine avoids relying on browser text layout for the page itself; CI runs golden-render tests on all three platforms; fall back to bundling a fixed webview on Linux if drift appears.
