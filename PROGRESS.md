## Workstream A — Locked pages + A-scenes ✅ (no unsound piece shipped)

The paginator's page-assembly loop was restructured around a labeled `Flow` (every page now carries a printed label alongside its physical ordinal) and locked pagination attaches exactly where `ARCHITECTURE.md` prepared: **Lock** (`compute_lock`) materializes scene numbers into the document, paginates freely once, and freezes each page's start as an anchor — `(scene number, element offset, non-whitespace char offset)` — stored in `project.json`. While locked, `apply_locked_anchors` re-cuts the block list at those anchors before flow: content between anchors fills its frozen page and overflow continues as A/B pages (`12` → `12A` → `12B`, base-26 letters), deletions leave short pages, and anchors that fell mid-speech at lock time re-emit the `(MORE)`/`NAME (CONT'D)` split at the recorded offset, so locked pages compose correctly with dialogue splitting (test-enforced) and with revision marks (the revised-line pass is orthogonal; also test-enforced). A-scene numbering (`12` → `12A`, collision-skipping) derives deterministically in `assign_scene_numbers`; **OMITTED** is a new `ElementKind` serialized as `.OMITTED #12#` in the Fountain superset and `Omitted="Yes"` in FDX (both round-trip tested), and an editor `lockGuard` plugin re-materializes an OMITTED placeholder whenever a whole locked scene is deleted (bundled into the same undo step). Golden tests: lock-freeze, edit-overflow → `1, 1A, 2` without reflowing later pages, deletion → short pages with stable labels, inserted-scene A-numbering (`1A`, then `1B`), omit rendering with margin numbers, unlock/relock sequential renumbering, and the dialogue-split/revision composition case. UI: Lock/Unlock in the Revisions panel behind a consequences-explaining confirmation (with milestone backup), locked badge in the status bar, frozen labels in the PDF header and editor page-break rules, amber A-page indicators and per-scene Omit in the navigator. One documented semantic (not a hack, a design choice): anchors are element-granularity with nonws-offset refinement — if the user deletes a locked scene's heading *and* its OMITTED placeholder in separate operations bypassing the guard, that page label drops rather than producing a blank page; the supported path is Omit, which preserves numbering. 62 Rust + 12 TS tests green.

## Workstream 5 — CRDT groundwork (Loro) ✅ (scoped; constraint documented)

Loro (Rust, v1.13) is now embedded in the core as the persistent history layer. Every save diffs the new Fountain serialization into a `LoroDoc` text container (`LoroText::update` turns each save into real CRDT text operations) and persists the binary state atomically as `script.loro` / `drafts/<name>.loro`; named versions additionally copy the CRDT state next to the snapshot file, so future history features can replay per-save edits and two copies of a project can merge deterministically. The invariant *fountain → CRDT → fountain is byte-identical* is test-enforced, plus tests for incremental history accumulation (`len_changes ≥ saves`) and corrupt-state recovery (a bad `.loro` restarts history fresh rather than failing the save — the fountain file remains the single source of truth, and CRDT write failures never block a save).

**Constraint hit, and the deliberate scope decision:** the prompt's "authoritative in-memory document store in the core, edits apply as CRDT operations" would require either (a) per-keystroke IPC from the ProseMirror editor to a stateful Rust store — putting a serialization round-trip on the keystroke path, directly violating the latency constraint — or (b) a WASM build of Loro inside the webview plus a second source of truth to reconcile, a data-safety risk. Both were rejected per the escape hatch. What shipped instead: CRDT ops at *save granularity* (debounced 2 s, blur, close — never a keystroke), which preserves the spec's actual goal (the document model carries CRDT history from v1 so collaboration isn't a retrofit). Options if finer granularity is wanted later: (1) batch ProseMirror steps and flush them to the Rust store on idle, (2) run Loro-WASM in a worker thread mirroring steps off-path, (3) move text ownership fully into Rust when the editor is eventually virtualized. 49 Rust tests green.
